## 2. 제안 구조 vs 현재 구조 GAP 분석

### Level 1: Daily Work Log (일일 업무일지)

**제안 구조** (캠스 현행 + 구조화):
```
1. 금일 수행 업무 (업무명, 세부내용, 진행상태)
2. 진행 중 업무 (업무, 현재 진행률, 다음 액션)
3. 이슈 / 문제 (발생 문제, 원인, 필요 지원)
4. 익일 계획 (예정 작업, 목표)
5. 특이사항 / 건의
```

**현재 상태**:
- WorklogQuickNew = 자유형 HTML 에디터 (Quill)
- Initiative 선택 → 진행률 + 소요시간 + 메모 형식
- 1건의 업무일지 = 1개 Initiative에 대한 기록
- **강점**: OKR 연동, 진행률 추적, 시간 추적, 문서 태그
- **약점**: "금일업무/진행중/이슈/익일계획" 같은 구조화된 섹션 없음

**GAP**: 
- 현재는 Initiative 단위의 단건 기록 → 하루 전체를 조망하는 일일 보고서 형태가 아님
- 구조화된 섹션(이슈, 익일계획 등)이 없어 "기록"은 되지만 "관리"가 안 됨
- **해결 방안**: 
  - A안: WorklogQuickNew에 구조화 옵션 추가 (현행 유지하면서 확장)
  - B안: "일일 업무 요약" 페이지 신규 생성 (당일 Worklog들을 자동 집계 + 추가 입력)

### Level 2: Weekly Status Report (주간 리포트)

**제안 구조**:
```
1. 이번 주 주요 완료 업무
2. 진행 중 핵심 프로젝트
3. 문제 / 리스크
4. 다음 주 계획
5. 지원 요청 사항
```

**현재 상태**:
- WorklogAi: 기간 지정 → AI가 업무일지 요약 (on-demand)
- WorklogEvalMonthly: 월간 평가 리포트 (관리자 전용)
- **주간 리포트 전용 페이지 없음**

**GAP**:
- AI 요약은 있지만, 구조화된 Weekly Report 형태가 아님
- 팀장이 경영층에 보고할 수 있는 정형화된 포맷 없음
- **해결 방안**:
  - 주간 리포트 페이지 신규 생성
  - 해당 주의 Worklog 자동 집계 → AI가 5개 섹션으로 구조화
  - 팀장이 편집/확정 후 경영층에 공유

### Level 3: Work Manual (업무 매뉴얼)

**현재 상태**: ✅ 대부분 구현됨
```
Phase 1: 제조업 표준 템플릿 입력 (6개 섹션)
  1. 업무 정의  2. 업무 흐름  3. 관련 데이터 정의
  4. 데이터 저장 방식  5. ERP 화면 설명  6. 예외/담당/관련문서
Phase 2: StepFormEditor + AI 검증 + 완성도 측정
Phase 3: 검토 승인 + AI BPMN 프로세스 생성
```

**GAP**: 
- 제안 구조와 현재 구조가 거의 일치
- 남은 과제: 매뉴얼 공유(visibility), 버전 diff, 템플릿 라이브러리

## 3. 연결 구조 (Knowledge Flow)

제안된 3단 구조의 핵심은 **데이터 흐름**입니다:

```
[Level 1] Daily Work Log
    │  개별 업무 기록 (Initiative별)
    │  + 이슈/블로커/진행률
    │
    ├──→ [자동 집계] ──→ [Level 2] Weekly Status Report
    │                        │  AI가 주간 Worklog를 요약
    │                        │  팀장이 편집/확정/보고
    │                        │
    │                        └──→ 경영층 리포트
    │
    └──→ [패턴 인식] ──→ [Level 3] Work Manual
                             │  반복 업무를 AI가 감지
                             │  매뉴얼 초안 자동 제안
                             │
                             └──→ BPMN 프로세스 템플릿
```

### 현재 구현된 연결:
- ✅ Worklog → Initiative → KR → Objective (OKR 역방향 추적)
- ✅ Work Manual → AI → BPMN Process Template (매뉴얼→프로세스)
- ✅ Process Task → Worklog (프로세스 실행 중 업무일지 작성)
- ✅ DocumentTags (품번/금형/차종/협력사/설비)

### 미구현 연결:
- ❌ Daily Worklogs → Weekly Report (자동 집계)
- ❌ Daily Worklogs → Manual 초안 제안 (패턴 인식)
- ❌ 자유형 태그 (#사출프로세스 #ERP #설비점검) → Knowledge DB

## 4. 구현 로드맵 (우선순위)

### Phase A: 업무일지 구조화 (1주)
> 현행 유지하면서 구조화 옵션 추가

| 항목 | 내용 | 난이도 |
|------|------|--------|
| A-1 | WorklogQuickNew에 "일일 보고서 모드" 토글 추가 | 중 |
| | → 금일업무 / 진행중 / 이슈 / 익일계획 / 특이사항 섹션 | |
| A-2 | Worklog 모델에 structuredData JSON 필드 추가 | 저 |
| | → 구조화된 데이터를 별도 저장 (기존 note 호환 유지) | |
| A-3 | WorklogDocument에 구조화 렌더링 추가 | 저 |
| A-4 | 기존 자유형 모드는 그대로 유지 (하위호환) | - |

### Phase B: 주간 리포트 (1~2주)
> 신규 기능

| 항목 | 내용 | 난이도 |
|------|------|--------|
| B-1 | WeeklyReport 모델 생성 (Prisma) | 중 |
| | → userId, orgUnitId, weekStart, sections(JSON), status | |
| B-2 | weekly-reports.controller.ts 백엔드 API | 중 |
| | → CRUD + AI 주간 요약 엔드포인트 | |
| B-3 | WeeklyReport.tsx 페이지 신규 | 고 |
| | → 주간 Worklog 자동 집계 → AI 구조화 → 편집/확정 | |
| B-4 | 네비게이션에 "주간 리포트" 메뉴 추가 | 저 |

### Phase C: Knowledge 태그 시스템 (1주)

| 항목 | 내용 | 난이도 |
|------|------|--------|
| C-1 | 자유형 해시태그 입력 추가 (#사출 #ERP #설비점검) | 중 |
| | → Worklog, WorkManual 모두에 적용 | |
| C-2 | Tag 모델 또는 JSON 배열 필드 추가 | 저 |
| C-3 | 태그 기반 검색/필터 (WorklogSearch 확장) | 중 |
| C-4 | 태그 클라우드 / 자동완성 | 저 |

### Phase D: 연결 자동화 (2주)
> AI 기반 자동 연결

| 항목 | 내용 | 난이도 |
|------|------|--------|
| D-1 | 주간 리포트 AI 자동 생성 | 중 |
| | → 해당 주 Worklog를 5개 섹션으로 구조화 | |
| D-2 | 반복 업무 감지 → 매뉴얼 초안 제안 | 고 |
| | → 같은 태그/Initiative의 Worklog 패턴 분석 | |

## 6. 기존 강점 활용 포인트

현재 서비스에서 이미 잘 갖추어진 부분:

1. **OKR 연동**: Worklog → Initiative → KR → Objective
   - 업무일지가 단순 기록이 아닌 목표 달성 추적으로 연결됨
   - 주간 리포트에서 OKR 진척도 자동 표시 가능

2. **프로세스 실행 연계**: Process Task → Worklog
   - 매뉴얼에서 생성된 프로세스가 실제 업무일지와 연결
   - Level 3 → Level 1 순환 구조 이미 존재

3. **AI 인프라**: OpenAI API + 프롬프트 파이프라인
   - 업무일지 AI 요약 (WorklogAi) 이미 동작
   - 매뉴얼 AI 검증/BPMN 생성 이미 동작
   - 주간 리포트 AI 생성에 동일 인프라 재사용 가능

4. **제조업 특화 태그**: 품번/금형/차종/협력사/설비
   - Knowledge DB의 기초 인프라가 이미 있음
   - 자유형 해시태그만 추가하면 확장 가능

## 7. 의사결정 필요 사항

1. **업무일지 구조화 방식**: 
   - A안: 기존 WorklogQuickNew에 "구조화 모드" 추가 (기존 UI 확장)
   - B안: 별도 "일일 보고서" 페이지 신규 생성

2. **주간 리포트 대상**: 
   - 팀장만 작성? 전 직원 작성?
   - AI 자동 생성 후 편집? 수동 작성?

3. **태그 확장 범위**: 
   - 기존 5종 마스터 태그 유지 + 자유형 해시태그 추가?
   - 자유형으로 통합?

4. **구현 우선순위**: 
   - Phase A (업무일지 구조화) 먼저?
   - Phase B (주간 리포트) 먼저?
