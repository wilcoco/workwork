# 경영지시 처리 업무 매뉴얼

## 1. 목적
경영층/리더의 지시가 구두·메모로 흩어져 유실되는 것을 막고, 지시 → 실행 → 기록 → 검수가 한 흐름으로 남도록 한다. 모든 지시는 Worklog의 **경영지시** 기능으로 등록하고, 실행 내용은 **업무일지**로 축적하며, 완료는 **지시자 검수**로 확정한다.

## 2. 적용 범위
- 대상: 전 구성원 (지시자 = 팀장 이상/경영층, 담당자 = 지시받은 구성원)
- 시스템 메뉴: [경영지시] ▸ 지시 등록 / 지시 목록, 홈 화면 "나에게 온 업무 지시", [목표관리] ▸ 중점 추진 과제

## 3. 용어
- **꼭지**: 지시를 실행 가능한 3~6개의 굵직한 매듭으로 나눈 단위. 순서대로 진행된다.
- **자동 담당 지정**: 지시문에 구성원 실명이 담당자로 거론되면 해당 꼭지의 담당자로 자동 지정된다. (동명이인은 자동 지정되지 않으므로 직접 배정)

---

### STEP S1 | 지시 등록
- taskType: WORKLOG
- 담당: 지시자 (팀장 이상/경영층)
- 목적: 지시를 시스템에 캡처하여 유실을 방지한다.
- 방법: [경영지시] ▸ [지시 등록]에서 평소 말하듯 자연어로 입력한다. 담당시킬 사람이 정해져 있으면 **실명을 문장에 포함**한다. (예: "현황 정리는 홍길동이 이번 주까지")
- 산출물: 등록된 지시 1건 + AI가 분해한 실행 꼭지 3~6개
- 완료조건: 지시가 등록되고 꼭지 분해 결과가 표시됨

### STEP S2 | 꼭지 검토 및 담당 배정
- taskType: WORKLOG
- 담당: 지시자
- 목적: AI 분해 결과가 실제 실행 순서와 맞는지 확인하고 담당을 확정한다.
- 방법:
  - 꼭지가 엉뚱하게 나뉘었으면 [재분해] 실행 (이름 거론자 자동 지정도 다시 적용됨)
  - 이름이 거론된 꼭지는 담당자가 자동 지정되어 있음 — 맞는지 확인
  - 담당이 비어 있는 꼭지는 직접 배정하고 필요 시 기한(마감일)을 지정
- 완료조건: 첫 번째 꼭지가 활성(ACTIVE) 상태이고 담당자가 지정됨
- 기한: 지시 등록 후 1일 이내

### STEP S3 | 담당 업무 수행 및 업무일지 기록
- taskType: WORKLOG
- 담당: 해당 꼭지 담당자
- 목적: 지시받은 꼭지를 실행하고 진행 내용을 기록으로 남긴다.
- 방법:
  - 배정되면 홈 화면 "나에게 온 업무 지시"에 [경영지시] 항목이 뜨고, [목표관리] ▸ [중점 추진 과제]에 과제가 자동 생성됨
  - 업무를 진행하며 **업무일지 작성 시 해당 중점 추진 과제를 연결**해 진행 내용을 기록
  - 진행이 막히면(블로킹) 지시자에게 즉시 공유
- 산출물: 과제에 연결된 업무일지(진행 기록), 꼭지의 기대 결과물
- 완료조건: 꼭지의 기대 결과가 달성되고 근거 일지가 기록됨
- 기한: 꼭지에 지정된 마감일 (미지정 시 지시자와 협의)

### STEP S4 | 완료 제출
- taskType: WORKLOG
- 담당: 해당 꼭지 담당자
- 목적: 꼭지 완료를 지시자에게 공식 제출한다.
- 방법: [경영지시] ▸ 해당 지시 상세 ▸ 꼭지의 [완료 제출]. 결과 요약과 근거(일지·산출물)를 함께 남긴다.
- 완료조건: 꼭지 상태가 검수(REVIEW)로 전환되고 지시자에게 검수 요청이 전달됨

### STEP S5 | 지시자 검수
- taskType: APPROVAL
- 담당: 지시자 (원 지시를 내린 경영층/리더)
- 목적: 결과가 지시 의도와 기대 결과에 부합하는지 판정한다.
- 방법: 제출된 결과와 연결된 업무일지를 확인 후 승인 또는 반려
- 분기:
  - **승인** 시 → STEP S6으로 진행
  - **반려** 시 → STEP S3으로 되돌아가 담당자가 보완 후 재제출 (반려 사유 필수 기재)
- 완료조건: 승인 또는 반려 판정 완료
- 기한: 제출 후 2일 이내

### STEP S6 | 다음 꼭지 진행 또는 지시 종결
- taskType: WORKLOG
- 담당: 다음 꼭지 담당자 (없으면 지시자)
- 목적: 승인된 꼭지 다음의 실행을 끊김 없이 잇는다.
- 방법:
  - 승인되면 다음 꼭지가 자동 활성화되고, 담당자에게 업무 지시·알림이 자동 전달됨 → 해당 담당자는 STEP S3부터 반복
  - 마지막 꼭지가 승인되면 지시 전체를 종결하고, 지시 목록에서 완료 상태를 확인
- 완료조건: 모든 꼭지 완료(지시 종결) 또는 다음 꼭지 실행 개시

---

## 4. 예외 처리
- **담당자가 자동 지정되지 않은 경우** (이름 미거론·동명이인): 지시자가 STEP S2에서 직접 배정한다.
- **담당자가 기한 내 착수하지 않는 경우**: 시스템 넛지(독촉 알림)를 활용하고, 지속 지연 시 지시자가 담당을 교체한다.
- **지시 내용이 변경된 경우**: 진행된 꼭지가 없으면 [재분해], 이미 진행 중이면 꼭지 수정 또는 새 지시로 등록한다.

## 5. 기록과 책임
- 모든 지시·배정·제출·검수 이력은 시스템에 자동 기록된다.
- 지시의 최종 책임은 지시자에게, 각 꼭지의 실행 책임은 배정된 담당자에게 있다.
