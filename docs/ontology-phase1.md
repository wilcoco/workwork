# 온톨로지 1단계: 활동(Activity) 등록부 — 구조와 테스트

## 무엇이 생겼나
- **Activity 등록부**: 회사 작업의 단일 사전 (이름/유형/담당힌트/판단기준/별칭)
- **자동 정합**: 프로세스 템플릿 저장 시 태스크 노드를 사전과 대조
  - 결정론(정규화·별칭 일치) → AI 일괄판정(유사 후보만, 보수적) → 신규 등록
- **연결**: BPMN노드·ProcessTaskTemplate·Worklog(+KeyResult/KeyInitiative 컬럼 준비)
- **지식 귀속**: 프로세스 태스크로 쓴 일지가 활동에 연결 → 🏅인증 지식이 활동에 축적
- **활용**: 일지 작성 화면 "이 작업의 과거 지식" / AI 보완질문이 기존 지식·기준을 알고 질문 / AI 답변이 활동 지식 우선 인용
- **대시보드**: 프로세스 관리 ▸ 회사 활동 지도 (요약·지식공백 리스크·지식자산·활동 목록·지식 모달)

## 오프라인 검증 (완료: 18/18)
정규화/결정론 매칭/별칭/유사후보 컷오프 + mock DB 샘플 흐름:
첫 저장=활동 3개 신규 → 다른 표기 재저장=기존 재사용(신규 1개만) → 재저장 멱등.

## 배포 후 실사용 테스트 시나리오
1. [매뉴얼] 내 매뉴얼 입력 → "구매 발주 처리" 표준양식으로 작성 → 프로세스 만들기 → 템플릿 완성
2. [지도] 프로세스 관리 ▸ 회사 활동 지도 → 발주요청서 작성/발주 승인/입고 확인 활동 자동 등록 확인
3. [재사용] 두 번째 매뉴얼(예: "긴급 발주")에 "발주요청서 작성" 포함 → 완성 → 지도에서 해당 활동의 '프로세스 사용' 2로 증가(+별칭 확인)
4. [실행·지식] 프로세스 시작 → 태스크에서 업무일지 작성(원인/재발방지 포함) → 🏅인증 → 지도에서 그 활동의 지식 1 증가, 활동 클릭 시 지식 표시
5. [재사용 지식] 같은 태스크로 다른 사람이 일지 작성 진입 → 상단 "이 작업의 과거 지식" 패널 확인
6. [AI] 업무 자료 분석에 해당 작업 질문 → [★AI 지식인증] 인용 확인
