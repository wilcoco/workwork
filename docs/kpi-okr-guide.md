# KPI · OKR 구조와 사용 가이드

> 대상: 전 구성원 (특히 팀장, 신규 입사자)
> 이 문서는 "정성 목표 OKR을 팀장이 팀 KR로 전개하면 담당자들이 수치를 입력해서 KPI 지수에 반영되나요?"라는 질문에서 출발했습니다. **답은 "아니오"이며, 왜 아니오인지가 이 시스템의 핵심 사상입니다.**

---

## 0. 질문에 대한 직답

**정성 목표(OKR)는 KPI 지수에 반영되지 않습니다. 둘은 처음부터 다른 트랙입니다.**

- **수치를 입력해서 지수가 올라가는 것 = 정량 목표(팀 KPI)** 트랙입니다.
  팀 KPI를 팀장이 팀원에게 분배하면, 담당자가 **업무일지에서 수치를 입력**하고 그게 KPI에 자동 집계됩니다.
- **정성 목표(OKR)** 는 수치가 아니라 **"무엇을 해냈는가"라는 내용(컨텐츠)** 이 쌓이는 트랙입니다. 중점 추진 과제와 업무일지의 진행 내용이 OKR 아래로 정렬되어 축적됩니다.

즉, "팀장이 전개 → 담당자가 입력"이라는 **동작은 두 트랙 모두 같지만, 입력하는 것이 다릅니다**: KPI에는 **숫자**를, OKR에는 **진행 내용**을 입력합니다.

---

## 1. 왜 둘을 분리했는가 (취지)

회사 업무는 성격이 다른 두 종류가 섞여 있습니다. 하나의 잣대로 재면 반드시 왜곡이 생깁니다.

| | **정량 목표 (팀 KPI)** | **정성 목표 (OKR)** |
|---|---|---|
| 업무 성격 | **오퍼레이션(반복 운영) 업무** — 생산, 품질, 납기, 비용처럼 매일 돌아가는 일 | **프로젝트(변화·개선) 업무** — 신규 개발, 개선 과제, 도전적 목표 |
| 묻는 질문 | "**얼마나 효율적으로** 돌리고 있나?" (효율성) | "**의미 있는 변화를** 만들어냈나?" (효과성) |
| 측정 방식 | **숫자** — 목표치 대비 실적치 | **내용** — 무엇을 시도했고 무엇이 달라졌는지의 기록 |
| 쌓이는 것 | 지표 실적 데이터 → 지수/추세 | 진행 컨텐츠(업무일지·과제 기록) → 회사의 지식 |
| 평가 관점 | 목표 달성률 (100%가 정상) | 도전의 크기와 진척 (70%도 훌륭할 수 있음) |

**분리하지 않으면 생기는 문제:**
1. 오퍼레이션 수치(예: 불량률)를 OKR처럼 다루면 → 매일 하는 일을 "목표"라고 다시 쓰는 형식적 문서 작업이 됨.
2. 프로젝트 목표(예: 신규 라인 안정화)를 KPI처럼 숫자 하나로 다루면 → 과정에서 얻은 지식이 아무 데도 남지 않고, 숫자를 맞추기 위한 목표 하향이 일어남.
3. 도전적인 일을 맡은 사람이 수치 평가에서 불리해져 **아무도 어려운 일을 맡지 않으려 함.**

그래서 이 시스템은 **효율(KPI)과 효과(OKR)를 분리해서, 각자에게 맞는 입력·측정·평가 방식**을 붙였습니다.

## 2. 정량 목표 (팀 KPI) — 숫자가 흐르는 트랙

### 구조
```
팀 KPI 지표 (팀 단위 목표치)
   └─ 개인 분배 (팀장이 팀원별로 할당)
        └─ 담당자가 업무일지에서 실적 수치 입력
             └─ KPI 자동 집계 → 리포트/현황판 반영
```

### 사용 방법

**[팀장] ① 팀 KPI 지표 만들기** — 메뉴: `목표관리 > 정량 목표 (팀 KPI) 입력`
- 팀이 관리할 지표를 등록합니다. 예: "조립 불량률", "납기 준수율", "월 생산량"
- 지표마다 목표치·단위·방향(이상/이하)을 정합니다.

**[팀장] ② 지표를 팀원에게 분배** — 같은 화면에서 지표별 담당자 지정
- 이 지표의 수치를 누가 입력할지(=현장에서 그 일을 하는 사람) 할당합니다.

**[담당자] ③ 업무일지 쓸 때 수치 입력** — 메뉴: `업무일지 작성`
- 일지 작성 화면에 **나에게 할당된 KPI 지표 입력란**이 나타납니다.
- 그날의 실적 수치를 적으면 끝입니다. 별도 화면에 갈 필요 없습니다.
- (일지 밖에서 몰아서 넣어야 할 때는 `목표관리 > KPI 실적 입력 (수동)` 사용)

**[모두] ④ 확인** — `정량 목표 (팀 KPI) 조회`, `팀 KPI 리포트`
- 입력된 수치가 자동 집계되어 목표 대비 실적, 추세를 보여줍니다.
- 임원/경영진은 `전사 목표 현황`에서 팀 KPI를 클릭하면 **누가 얼마를 입력했는지**(개인 기여, 입력 이력)까지 내려가 볼 수 있습니다.

### 핵심 규칙
- **KPI 수치는 업무일지를 통해서 들어오는 것이 기본입니다.** "일하면서 기록하고, 기록하면 지표가 된다" — 지표 입력을 위한 별도 업무를 만들지 않는 것이 설계 의도입니다.

## 3. 정성 목표 (OKR) — 내용이 쌓이는 트랙

### 구조
```
Objective (목표: 어떤 변화를 만들 것인가)
   └─ Key Result (핵심 결과: 그 변화가 일어났다는 증거)
        └─ 중점 추진 과제 (KR을 실제로 밀고 가는 프로젝트/과제)
             └─ 업무일지·과제 진행 기록 (실행 컨텐츠가 여기 축적됨)
```

### 사용 방법

**[팀장/리더] ① O와 KR 입력** — 메뉴: `목표관리 > 정성 목표 (OKR) 입력`
- Objective: 방향 문장. 예: "함평공장 도장 품질을 고객 클레임 제로 수준으로 끌어올린다"
- Key Result: 그것이 이루어졌음을 보여줄 결과. 예: "재도장률 절반 감축", "클레임 재발 0건 프로세스 정착"
- ⚠️ KR에 숫자가 들어갈 수는 있지만, **이 숫자는 KPI 지수 집계에 들어가지 않습니다.** KR의 숫자는 "성공의 정의"이고, KPI의 숫자는 "매일의 실적"입니다.

**[팀장→담당자] ② 중점 추진 과제로 전개** — 메뉴: `목표관리 > 중점 추진 과제`
- KR을 달성하기 위한 실제 과제를 만들고 담당자를 지정합니다.
- 과제를 만들 때 **어느 OKR(Objective)에 정렬되는지** 선택합니다. (돌발 과제는 정렬 없이 독립으로도 가능)

**[담당자] ③ 진행 내용 기록**
- 과제를 진행하며 **업무일지**를 쓰면, 일지에서 해당 중점과제를 연결해 **진행 내용이 과제에 컨텐츠로 축적**됩니다.
- 수치가 아니라 "무엇을 했고, 무엇을 알게 됐고, 다음에 무엇을 할지"가 기록됩니다.

**[모두] ④ 확인** — `정성 목표 (OKR) 조회`
- O → KR → 과제 → 진행 기록으로 내려가며 "목표가 실제로 움직이고 있는지"를 내용으로 확인합니다.

### 핵심 규칙
- OKR의 성과물은 **축적된 컨텐츠**입니다. 분기가 끝나면 이 기록이 그대로 회사의 노하우(매뉴얼·프로세스의 재료)가 됩니다.

## 4. 실무 시나리오로 보는 구분

**시나리오 A — "이번 달 사출 불량률을 관리해야 한다"**
→ 반복 운영 업무 = **KPI 트랙**.
팀장이 `정량 목표 (팀 KPI) 입력`에서 지표 등록 → 담당자 할당 → 담당자는 매일 업무일지에 불량률 수치 입력 → 리포트에서 추세 확인.

**시나리오 B — "불량률을 구조적으로 낮추는 개선 프로젝트를 한다"**
→ 변화·개선 업무 = **OKR 트랙**.
`정성 목표 (OKR) 입력`에 O/KR 등록 → `중점 추진 과제`로 개선 과제 생성·담당자 지정 → 담당자는 업무일지에 실험/개선 내용 기록(과제에 축적) → `OKR 조회`에서 진척 확인.

**같은 "불량률"이라는 소재라도**: 매일의 관리는 KPI로(숫자), 구조를 바꾸는 도전은 OKR로(내용) 갑니다. **B 프로젝트가 성공하면 그 결과가 A의 숫자로 나타납니다** — 이것이 두 트랙의 올바른 관계입니다.

## 5. 자주 하는 혼동 Q&A

**Q1. 정성 목표(OKR)를 팀 KR로 전개하면 담당자가 수치를 입력해서 KPI에 반영되나요?**
→ 아니오. OKR 트랙에는 수치 입력→지수 반영 흐름이 없습니다. 수치 입력→자동 집계는 **정량 목표(팀 KPI)** 메뉴에서 만든 지표에만 있습니다. OKR에서 담당자가 입력하는 것은 **과제 진행 내용**입니다.

**Q2. 그럼 KR에 "불량률 30% 감축"이라고 숫자를 쓰면 안 되나요?**
→ 써도 됩니다. 다만 그 숫자는 "성공했는지 판단하는 기준"일 뿐, 매일 실적을 입력하는 대상이 아닙니다. 매일 입력·집계가 필요하면 같은 지표를 **팀 KPI로도** 등록하세요.

**Q3. 업무일지에 KPI 입력란이 안 보여요.**
→ 팀장이 그 지표를 본인에게 **할당**했는지 확인하세요. 할당된 지표만 일지에 나타납니다.

**Q4. 중점 추진 과제는 꼭 OKR에 연결해야 하나요?**
→ 아니오. 돌발성·독립 과제는 정렬 없이 만들 수 있습니다. 다만 분기 목표와 관련된 과제라면 정렬해 두어야 OKR 조회에서 함께 보입니다.

**Q5. 한 줄로 요약하면?**
→ **"숫자는 KPI로, 이야기는 OKR로."** 매일 돌아가는 일의 실적 숫자는 업무일지에서 KPI로 들어가고, 변화를 만드는 프로젝트의 진행 이야기는 중점과제를 통해 OKR에 쌓입니다.

## 6. 메뉴 빠른 참조

| 하고 싶은 일 | 메뉴 (목표관리) | 누가 |
|---|---|---|
| 팀 KPI 지표 등록·담당 분배 | 정량 목표 (팀 KPI) 입력 | 팀장 |
| KPI 실적 수치 입력 | **업무일지 작성** (기본) / KPI 실적 입력 (수동) | 담당자 |
| KPI 현황·추세 확인 | 정량 목표 (팀 KPI) 조회 · 팀 KPI 리포트 | 모두 |
| O/KR 등록 | 정성 목표 (OKR) 입력 | 팀장/리더 |
| 과제 생성·담당 지정·OKR 정렬 | 중점 추진 과제 | 팀장/담당자 |
| 과제 진행 내용 기록 | 업무일지 작성 (과제 연결) | 담당자 |
| OKR 진척 확인 | 정성 목표 (OKR) 조회 | 모두 |
| 전사 통합 현황 (KPI 개인 기여까지) | 전사 목표 현황 | 팀장 이상 |
